# 文件操作——创建、编辑与删除

> 在浏览器中直接管理仓库文件，无需本地环境即可完成基本编辑。

## 概述

GitHub 不仅是代码托管平台，它本身也是一个功能完善的 Web 编辑器。你可以在浏览器中直接创建、编辑、删除和上传文件，所有操作都会生成对应的 Commit 记录。这对于快速修复文档错误、添加配置文件或管理小型项目来说非常方便。

当然，对于大型项目和复杂编辑场景，本地 IDE 配合 Git 命令仍然是最优选择。但理解浏览器端的文件操作能力，能帮助你在任何设备上随时响应紧急需求。

> [!NOTE]
> 浏览器端编辑的文件大小上限为 25 MB。超过此限制的文件需要通过 Git 命令行或 GitHub CLI 上传。二进制文件（如图片、PDF）的上传上限为 100 MB。

本章将覆盖仓库文件管理的基本操作，帮助你快速上手浏览器端的文件编辑能力。

## 核心操作

### 创建新文件

1. 进入目标仓库，导航到你想创建文件的目录。
2. 点击 **Add file > Create new file**。
3. 在文件名输入框中输入文件路径和名称。
   - 输入 `/` 可以创建嵌套目录，例如 `docs/guide.md` 会自动创建 `docs` 目录。
4. 在编辑区输入文件内容。
5. 在页面底部填写 Commit 信息，选择 Commit 到当前分支还是新分支。
6. 点击 **Commit new file**。

```mermaid
flowchart LR
    A[进入仓库] --> B[Add file > Create new file]
    B --> C[输入文件名与路径]
    C --> D[编辑内容]
    D --> E[填写 Commit 信息]
    E --> F[Commit new file]
```

> [!TIP]
> 在文件名中输入 `docs/notes.md` 会自动创建 `docs` 目录并生成 `notes.md` 文件。这是在浏览器中创建空目录的技巧。

### 编辑已有文件

1. 进入目标仓库，导航到要编辑的文件。
2. 点击文件内容区域右上角的铅笔图标（Edit this file）。
3. 在编辑区修改内容。
4. 填写 Commit 信息，提交更改。

编辑器支持以下功能：
- **语法高亮**：根据文件扩展名自动识别语言。
- **换行模式**：Soft wrap（自动换行）和 No wrap（不换行）可切换。
- **差异预览**：点击 **Preview changes** 可以查看修改前后的对比。

### 删除文件

1. 进入目标仓库，导航到要删除的文件。
2. 点击文件内容区域右上角的垃圾桶图标（Delete this file）。
3. 填写 Commit 信息，提交删除操作。

> [!WARNING]
> 删除文件是不可逆的（虽然可以通过 `git revert` 恢复文件内容，但提交历史会保留删除记录）。删除前请确认该文件确实不再需要。

### 上传文件

1. 进入目标仓库，导航到目标目录。
2. 点击 **Add file > Upload files**。
3. 将文件拖拽到上传区域，或点击 **choose your files** 选择文件。
4. 填写 Commit 信息，点击 **Commit changes**。

上传限制：
- 单文件不超过 100 MB（通过浏览器上传）。
- 一次最多上传 100 个文件。
- 不支持上传整个文件夹（文件夹结构需要通过文件路径中的 `/` 来创建）。

### 使用 .gitkeep 保留空目录

Git 不会追踪空目录。如果你需要在仓库中保留一个空目录结构（例如 `logs/` 或 `uploads/`），可以在该目录中放置一个 `.gitkeep` 文件。

```bash
# 在本地创建空目录并添加 .gitkeep
mkdir -p logs
touch logs/.gitkeep
git add logs/.gitkeep
git commit -m "chore: add logs directory placeholder"
```

在浏览器中创建的方式：创建文件时将文件名设为 `logs/.gitkeep`，GitHub 会自动创建 `logs` 目录。

> [!NOTE]
> `.gitkeep` 并非 Git 的官方特性，而是社区约定俗成的惯例。它的唯一作用是让 Git 追踪该目录。你也可以使用 `.gitignore` 或 `README.md` 作为占位文件，效果相同。

## 进阶技巧

### 批量编辑多个文件

浏览器端每次只能编辑一个文件。如果需要同时修改多个文件，可以：

1. 使用 GitHub.dev 编辑器：在仓库页面按 **`.`**（句号键），会在浏览器中打开基于 VS Code 的 Web 编辑器。
2. 在 GitHub.dev 中可以同时编辑多个文件，所有修改会合并为一个 Commit 提交。

```mermaid
flowchart LR
    A[仓库页面] -->|按 . 键| B[GitHub.dev 编辑器]
    B --> C[多文件编辑]
    C --> D[源代码管理面板]
    D --> E[一次性 Commit]
```

> [!TIP]
> GitHub.dev 是一个轻量级的 Web IDE，支持语法高亮、多文件编辑、搜索替换等功能。它不需要安装任何软件，是快速编辑的理想选择。

### 从 URL 直接创建文件

GitHub 支持通过 URL 参数快速导航到创建文件页面：

```
https://github.com/<owner>/<repo>/new/<branch>/<path>
```

例如，要在 `main` 分支的 `docs` 目录下创建新文件，直接访问：

```
https://github.com/<owner>/<repo>/new/main/docs
```

### 使用 GitHub CLI 操作文件

```bash
# 通过 API 创建或更新文件
gh api repos/<owner>/<repo>/contents/<path> \
  -X PUT \
  -f message="<commit-message>" \
  -f content="$(base64 < <local-file>)" \
  -f branch="<branch>"

# 下载文件内容
gh api repos/<owner>/<repo>/contents/<path> \
  -q '.content' | base64 -d
```

### 在线预览文件

GitHub 支持直接在浏览器中预览多种文件类型：

| 文件类型 | 预览方式 |
|---------|---------|
| Markdown（.md） | 自动渲染为 HTML |
| 图片（.png/.jpg/.svg 等） | 内联显示 |
| PDF | 内置查看器 |
| CSV | 表格形式展示，支持搜索和排序 |
| GeoJSON/TopoJSON | 地图可视化 |
| STL | 3D 模型预览 |

## 常见问题

### Q: 浏览器编辑和本地编辑有什么区别？

浏览器编辑适合小幅修改（如修复文档错别字、更新配置值），每次修改会生成一个 Commit。本地编辑则支持批量操作、分支管理、冲突解决等高级功能。参见 [提交与提交历史](05-提交与提交历史) 了解 Commit 的最佳实践。

### Q: 上传文件时目录结构丢失怎么办？

浏览器上传不支持保留本地目录结构。解决方案：1）使用 Git 命令行 Push；2）在 GitHub.dev 中拖拽文件夹；3）使用 GitHub CLI 的 API 功能。

### Q: 如何恢复误删的文件？

如果文件刚被删除且尚未做其他 Commit，可以在提交历史中找到删除前的 Commit，点击文件内容中的 **Restore this file** 按钮。否则，通过 `git revert` 撤销对应的删除 Commit 即可恢复。

### Q: .gitkeep 和 .gitignore 有什么区别？

`.gitkeep` 是社区约定，用于让 Git 追踪空目录。`.gitignore` 是 Git 的官方配置文件，用于指定哪些文件不应被 Git 追踪。两者可以共存于同一目录。

### Q: 能否在浏览器中重命名文件？

可以。在编辑文件页面，修改文件名输入框中的路径即可实现重命名或移动文件。例如将 `old-name.md` 改为 `new-name.md`，或将 `file.md` 改为 `docs/file.md`（移动到 docs 目录）。

### Q: 上传大文件有什么限制？

浏览器上传的单文件上限为 100 MB。更大的文件需要使用 [Git Large File Storage (Git LFS)](https://git-lfs.github.com/)。GitHub 建议仓库总大小不超过 1 GB，强烈建议不超过 5 GB。
